🎓 EC2 Masterclass

Deep dive into Amazon EC2 using our Node.js User Service as the practical example. EC2 forms the backbone of AWS compute and is heavily tested across all AWS certifications.

Why EC2 Matters in Exams

Exams test your ability to select the most cost-effective and performant compute option. You must know when to use EC2 vs Serverless (Lambda), how to secure it, and how to scale it using ASGs and ELBs.

EC2 Naming Explained

Example: m7g.large

  • m = Instance Family (General Purpose)
  • 7 = Generation (newer is usually better/cheaper)
  • g = Additional capability (Graviton/ARM processor)
  • large = Instance Size (determines vCPU and RAM)

Instance Families

Prefix Family Name Best For
T, M, Mac General Purpose Web servers, microservices, code repos
C Compute Optimized High-perf web servers, batch processing
R, X, z Memory Optimized Databases, Redis caches, big data
I, D, H Storage Optimized Data warehousing, Elasticsearch, Kafka
P, G, Inf, Trn Accelerated Compute Machine learning, 3D graphics rendering

Pricing Models

Model When to use Exam Tip
On-Demand Unpredictable, short-term Most expensive, pay by second
Reserved (RI) Steady-state, 1-3 yr commit Can reserve capacity in a specific AZ
Savings Plans Consistent compute spend Applies across EC2, Fargate, Lambda
Spot Batch jobs, fault-tolerant Up to 90% off. Reclaimable w/ 2 min warning
Dedicated Strict licensing needs Physical server fully dedicated to you

Method Description

Lab: Deploying the Node.js User Service

Let's deploy our running example to a single EC2 instance manually to understand the underlying infrastructure.

1. Create Security Group

We need port 80 for NGINX and port 22 for initial SSH.

aws ec2 create-security-group \
  --group-name node-app-sg \
  --description "Node.js App"

# Allow HTTP
aws ec2 authorize-security-group-ingress \
  --group-name node-app-sg \
  --protocol tcp --port 80 \
  --cidr 0.0.0.0/0

2. Launch Instance with User Data

We use User Data to bootstrap the server and install dependencies.

aws ec2 run-instances \
  --image-id ami-0123456789 \
  --instance-type t3.micro \
  --security-groups node-app-sg \
  --user-data file://startup.sh

3. The startup.sh Script

#!/bin/bash
yum update -y
curl -fsSL https://rpm.nodesource.com/setup_18.x | bash -
yum install -y nodejs nginx
npm install -g pm2

mkdir -p /app && cd /app
cat <EOF > server.js
const express = require('express');
const app = express();
app.get('/users', (req, res) => res.json([{id: 1, name: "Alice"}]));
app.listen(3000);
EOF

npm install express
pm2 start server.js --name "user-service"

# Configure NGINX reverse proxy (port 80 -> 3000) inside nginx.conf...
systemctl start nginx && systemctl enable nginx

Troubleshooting & Exam Reminders

  • Timeout: Usually Security Group (port 80/443 closed) or missing Public IP.
  • Connection refused: OS firewall blocking, or web server (NGINX/Node) not listening on 0.0.0.0.
  • Security Groups are stateful. NACLs are stateless.
  • By default, root EBS volumes are deleted when an instance is terminated (`DeleteOnTermination=true`).
  • Stopping an instance keeps EBS data but loses Instance Store (ephemeral) data.
