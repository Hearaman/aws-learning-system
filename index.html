Lab: Node.js User Service

# 1. SG: Inbound 80 (HTTP)
aws ec2 authorize-security-group-ingress --group-name app-sg --protocol tcp --port 80 --cidr 0.0.0.0/0
# 2. Launch with User Data
aws ec2 run-instances --image-id ami-123 --instance-type t3.micro --user-data file://script.sh
# Inside script.sh:
yum install -y nodejs nginx
npm install -g pm2
pm2 start app.js --name "user-service"
systemctl start nginx

Exam Traps & Troubleshooting

  • Trap: Root volumes are deleted on termination by default. Instance store is lost on stop.
  • Trap: Changing instance type requires stopping the instance.
  • Ports: 22 (SSH), 80/443 (HTTP/S), 3000 (Node app), 5432 (RDS).
  • Timeout: Check Security Groups.
  • Refused: Check app/OS firewall.

🎓 VPC Masterclass

Deep dive into Amazon VPC using the same Node.js User Service architecture.

Core Building Blocks

ComponentPurpose
VPC CIDRDefines your private IP range, e.g. 10.0.0.0/16
Public SubnetHas a route to an Internet Gateway
Private SubnetNo direct internet route; often uses NAT for egress
Route TableControls where subnet traffic goes
Security GroupStateful instance or ENI firewall
NACLStateless subnet firewall

Best-Practice Layout

  • Put the ALB in two public subnets across two AZs.
  • Put EC2 app servers in private subnets.
  • Put RDS in private DB subnets with no public access.
  • Use NAT Gateway only for private subnet outbound internet access.
  • Use VPC Endpoints for private access to services like S3 and SSM.

Hands-On Lab: Node.js VPC

# Public ALB, private app, private DB
resource "aws_vpc" "main" { cidr_block = "10.0.0.0/16" }
resource "aws_subnet" "public_a"  { vpc_id = aws_vpc.main.id cidr_block = "10.0.1.0/24" }
resource "aws_subnet" "private_a" { vpc_id = aws_vpc.main.id cidr_block = "10.0.11.0/24" }
# Route 0.0.0.0/0 from public subnets to IGW
# Route 0.0.0.0/0 from private subnets to NAT Gateway
# EC2 SG allows 3000 only from ALB SG
# RDS SG allows 5432 only from EC2 SG

Exam Traps

  • Public subnet means route to IGW, not just “has a public IP”.
  • Security Groups are stateful; NACLs are stateless.
  • NAT Gateway enables outbound internet from private subnets only.
  • VPC Peering is non-transitive; Transit Gateway solves hub-and-spoke networking.
  • Site-to-Site VPN is encrypted and fast to set up; Direct Connect is private but not encrypted by default.
